# Taller de grafana 
El taller de grafana es una clase teorico practica de maximo 60 minutos para la catedra de ingenieria y calidad de software. 

## Objetivo del taller
Introduccion a la observabilidad, practica sobre grafana en ambito local. 
Dar las herramientas necesarias para que los alumnos apliquen una capa de observabilida a sus trabajo integradores y adquieran la competencia profesional de la observabilidad en la ingenieria de software. 

## Objetivos particulares
-Crear un entorno simple estilo "Hola Mundo" con el stack tecnologico aplicada en el trabajo integrador de la materia. Este stack en el tiempo puede cambiar.

### Stack tecnologico 2026
- Monorepo utilizando Typescript, React (Vite + Chakra UI) en el frontend, y Fastify con Prisma (PostgreSQL) en el backend, siguiendo los principios de la Arquitectura Hexagonal. Docker + multistage + Docker compose

### Idea de clase
**Teorica mediante presentacion de slide con MARP md**
- Defincion de Observabilida en sentido teorico con abordaje de Ingenieria de software
- Conceptos clave, terminos de la industria, standares y buenas practicas. Tendencias en observabilidad. 
- Abordaje a Grafana. Conceptos, stack, proyectos, docs y capacitacion. 
- "ABC" o intro al uso de grafana.
**Laboratorio en aula** 
- Crear un dasarrollo simple de un CRUD funcional sobre docker con identico stack al trabajo integrador. Estilo "Hola Mundo"
- Integrar grafana, desplegando en docker
- Conectar fuente de datos para ingesta de logs, metricas y trazas de frontend, backend y base de datos. 
- Crear un dashboard simple para mostrar datos concretos como pueden ser Request, Response, Recursos consumidos por los containers, y metricas clave de la BBDD.
